The app can use a key configured on the local server or a personal key entered for this browser session.

Personal-use mode: the key stays only in page memory and is sent to this app’s local server with each AI request. Closing or refreshing the page forgets it. For a shared or public deployment, use the server environment variable instead.

Mobile standalone setup

When installed from HTTPS, EKG Lens AI opens like an app from the home screen. Camera access requires HTTPS or localhost; the image upload control can still open the phone camera on many devices.

Limits: The discussion tool does not prescribe medications, provide patient-specific doses, or replace a 12-lead ECG reviewed with symptoms, serial tracings, labs, and examination.
Privacy and safety: The included server does not write ECG images or session API keys to disk. With AI enabled, the image, clinical context, and request are sent to OpenAI. A key entered in the page is retained only in memory until refresh or close. Do not upload identifying patient information unless the deployment is configured for applicable privacy and compliance requirements.
